Architecture:

Dragonfly Reader:

/server
--TelnetServer				| Server Handling Telnet Connections

/client
--TelnetClient				| Connected Client

/service
--AuthenticationService		| Handles login authentication
--MainLobbyService			| Main lobby for the Dragonfly Mud-Hub
--HelpService				| Help screens
--StoryService				| Executes the Story Game loop

/rendering
--TelnetRenderer			| Handles input and output on the socket

Dragonfly Data Service:

/controller
--RESTController			| Abstract REST API
--ChapterRESTController		| Chapter API (contains a map of pages)
--PageRESTController		| Page API (contains a map of entities)
--EntityRESTController		| Entity API (objects within the game)
--ComponentRESTController	| Component API (attached to entities to provide functionality)

/service
--SQLiteDataService			| Abstract Data Service for SQLite
--ChapterDataService		| Chapter DAS
--PageDataService			| Page DAS
--EntityDataService			| Entity DAS
--ComponentDataService		| Component DAS

/model
--Chapter					| Chapter DO
--Page						| Page DO
--Entity					| Entity DO
--Component					| Component Interface DO
--CDoorway					| Component Doorway DO
--CPickup					| Compenent Pickup DO
